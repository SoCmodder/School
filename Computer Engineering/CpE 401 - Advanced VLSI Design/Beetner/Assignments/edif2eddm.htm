Synthesis of VHDL code for custom ASIC design

 

Note: Major portions of this text were taken directly from the ADK toolset manual

 

1) Develop and test VHDL code

The first step in developing your custom ASIC design is to develop and test your VHDL code using vcom and vsim.

 

2) Synthesizing VHDL using Spectrum (batch mode)

 

After your code works in simulation, you’re ready to try to synthesize it.  Synthesis is performed with Leonardo Spectrum.  The result of this synthesis will be an EDIF netlist and a vhdl file of the synthesized result that you can test using vsim.

 

To synthesize your code, enter the following script into a file in the same directory as your VHDL code.  The name of the scriptfile is not important.  Modify the text for the portions shown in red.  For example, an appropriate input file might be blah.vhd, an appropriate EDIF output file might be blah.edf, and an appropriate VHDL output file might be blah_syn.vhd (DO NOT give the output file the same name as your input file!).  Once you’ve completed this portion, enter the following command at the Unix command prompt:

 

            spectrum –file scriptname

 

where scriptname is the name of the file where you placed your script.  If everything went well, you should have no errors. 

 

Record the total number of gates used and expected system clock speed from the spectrum output.  The clock speed is only an estimate and will vary with the final layout.

 

Script:

set _xmp_enable_renoir FALSE

set vhdl_write_component_package FALSE

set vhdl_write_use_packages "library IEEE, adk; use IEEE.STD_LOGIC_1164.all; use adk.all;"

set edifout_power_ground_style_is_net TRUE

set max_fanout_load 14

set force_user_locad_values TRUE

load_library ami05_typ

read  { input VHDL files }

pre_optimize -common_logic -unused_logic -boundary -xor_comparator_optimize

pre_optimize -extract

pd

optimize .work.entity.architecture -target ami05_typ -macro -area -effort quick -hierarchy auto

optimize_timing .work.entity.architecture

report_area -cell_usage -all_leafs

set report_delay_slack_threshold 0

report_delay  -num_paths 1 -critical_paths -clock_frequency

auto_write -format EDIF -downto PRIMITIVES output_file

auto_write -format VHDL -downto PRIMITIVES output_file

 

 

3) Post-Synthesis Simulation

After running spectrum, verify your synthesized code runs as expected by running vcom and vsim on the VHDL file produced by spectrum.

 

To compile the synthesized VHDL code will require use of the ADK libraries.  Verify that you have the ADK library mapped by typing ‘vmap’ at Unix command prompt. If it does not show up in the mapped list, map the ADK library using the command: 

 

vmap adk /usr/local/adk2/lib/adk_vhdl

 

Double check that the ADK library is mapped by typing ‘vmap’ at prompt once again.

 

Once the ADK library is mapped, compile your synthesized VHDL code using vcom and simulate your testbench using vsim.  Your test-bench should still work fine.

 

Note: I have had some strange errors while synthesizing the generated VHDL code where new variables are added to the code and are named strangely (like “\[870_\”). vcom has a problem with these variable names.  This has only happened occasionally, but if it happens to you, you simply have to replace the strange variable name with an acceptable name (everywhere in the VHDL code) and then run vcom.

 

4) Creating a schematic for the synthesized part (EDDM schematic from EDIF):

The next step in generating the files needed by MOSIS is to produce an EDDM schematic from the EDIF file. This is done using the edif2eddm script. The edif2eddm script creates a ‘work’ folder where in saves the schematic and its associated files. So, it’s a better to generate this schematic in a different directory.

 

First, create a directory “netlist” (name is not important) and copy the .edf file to this directory

>mkdir netlist

>cd netlist

>cp ../design_name.edf design_name.edf

>mgc_csh

>edif2eddm design_name.edf

 

where design_name is the name of your design. There may be a few errors generated by the Schematic Generator option that you can ignore for now – there is a bug in the script that we will fix in the following step (step 5). The edif2eddm script will locate the top level of your design from the EDIF file, will create a ‘work’ directory in the current directory, and (theoretically) will put a new schematic in it. It creates a viewpoint with your design_name and additional viewpoints for IC layout using the adk_dve script.

 

5) Schematic Generator

To get around a bug in the edif2eddm script, we will need to generate the schematic manually.  First get into the right directory, then invoke the schematic generator tool:

 

>cd work

>swd

>sul

>sg

 

Use the File->Open Design from Viewpoint... menu item to bring up the Open Design from Viewpoint dialog box. Click on the Navigator... button and go into the design_name component. Select the  folder next to design_name and click OK in the Open Design from Viewpoint dialog box. (There should be 6 viewpoints in all: accusim, ami05, design_name, layout, lvs, sdl). A blank schematic window will appear in the sg tool window. If you get an error about viewpoint not found, quit sg, delete the work folder and run the edif2eddm script again.

 

Use the Setup->Symbol->Use Genlib Classification menu item to setup the symbol classifications and then use the Partition Setup... command from the pop up menus (right mouse button) to bring up the Partition Setup dialog box. Click on the Number of Sheets button, make sure the Number of Sheets item is set to 1 and click OK . Use the Generate item from the pop up menus to generate a schematic.  The schematic that comes up is a standard-cell layout of your design

 

Use the File->Save menu item to save the schematic and exit the sg tool.

 

6) Automated Layout using ICStation

Your next step is to create the layout of your overall design.

 

In your working directory (the one that contains the ‘synthesized’ schematic and its viewpoints – like netlist/work above... the same directory you ran sg in), invoke ICStation using

>sul

>swd

>ic (or adk_ic)

 

From the Session palette, select ‘Create’ [d1] under the “cell” heading and enter the following in the dialog box that appears.  You can use the navigator at times to make this easier – however, I was not able to get my design name to show up properly in the navigator and had to enter it manually:

            Cell Name: design_name

            Attach Library: $ADK/technology/ic/ami05

            Process: $ADK/technology/ic/ami05

            Rules File: $ADK/technology/ic/ami05.rules

            Angle Mode: 45

            Connectivity? : With Connectivity

            EDDM Schematic Viewpoint: design_name/layout

            Logic Loading Options…..

                                    Logic Loading: Flat

 

Leave all other options at default values and OK the dialog box. An empty cell window will appear with the design name.

 

(Note: if you screw up and have to re-create your cell, your best option right now is to create a new cell under a different name... we are still working on the best way of deleting your old cell without corrupting other parts of your design).

 

Floorplanning

From the IC Palettes menu (on the right hand side), click on Place & Route to goto the Place & Route palette and select Autofp.

In the Autofloorplan Options dialog box, allow all options to default simply by clicking on OK.

 

Use View à All or (Shift +F8) to see the automatically generated floor plan. You will see boxes enclosed by solid green bars along each edge. The boxes indicate the rows into which cells will be organized. The solid bars indicate edges of the cells along which physical ports will be placed.

Depending on the complexity of your design, you’ll have more than one row. If you have only one row for a complex design, something went wrong. Find it, fix it and re-do this step.

 

Standard Cell Placement

In the Place & Route palette, choose StdCel under Autoplc (autoplace standard cells). Click on OK in the dialog box.

You should see individual cells placed in the floorplan boxes. Cell locations are determined by their interconnectivity. Cells which share connections are placed near one another.

 

If you’d like to look inside the cells to see the transistor-level layout, select Peek on View in the Setup à IC dialog box.  If your design is large, you may have to zoom in to see the transistor-level layout.

 

Placing Ports

Choose Ports under Autoplc to place ports. You can allow the options to default for now or use the even option for even placement if you do not like the default placement. You will see lightly shaded blue or purple areas along the bars at the edge of the layout.  These are the ports to your design.

 

Find the relative location of the ports by scrolling about the edge of your design and selecting/deselecting the shaded areas with your mouse and the F2 (select/unselect) key. Routing

Now, from the Autorou section of the Place & Route palette select All, and accept the defaults in the dialog box. This should route all connections in our cell. You will see the cell rows move apart and additional metal runs appear in the channels between rows. Depending on the size of the design, autorouting may take several minutes.

 

Several small overflows may still exist. These small overflows may not be viewable on a zoomed out design. To select all overflows in the design type “check over” in the design window. In the form window that appears select All and OK the form. Next, from the Place & Route palette, select “Overflw” under Autorou. If the response “An object of type Overflow must be selected” appears in the status block, there are no overflows to route. Otherwise, the overflows should be routed (i.e. you need to find a way to round the metal or poly from one unconnected point – overflow – to the other).

 

Compacting

The autorouter’s objective is to complete as many connections in the design as is possible, with less regard for wasted area. For this reason, a layout compactor is available which “squashes” things together. To take advantage of this tool, from the PR Edit section of the Place & Route palette, select Compct. We want to compact in both the horizontal and vertical dimensions, so do this twice:

once down and then again in the left direction. Do not do this more than once along a given axis. Not only does it rarely result in improvement, it can lead to layout errors in some versions of the compactor.

 

Verify that after routing and compacting, the relative port positions have not changed (note: this may be more difficult than it sounds, as metal layers may now be routed over the ports.  You might use F3 – cycle selected – and F1 – select area – to help you find the port names or use right mouse>select>group>ports).  Write down the new port configuration.

 

Save File

It is a good idea to occasionally save your results during this process.  You may save your cell using the File>Cell>Save Cell option.

 

To edit the cell after saving, don’t forget that you have to reserve the cell.  You can reserve the cell using the File>Cell>Reserve option.

 

 

7) Logic Size

Now that the layout of your logic is complete, you can get an estimate of its overall size using the Other>Measure Distance tool.  Results are given in lambda (not microns). 

 

For the 0.5 micron AMIS process, MOSIS allows designs that are 1.5 mm by 1.5 mm, 1.5 mm by 3.0 mm, and 3.0 mm by 3.0 mm.  If your design won’t fit in this space, it’s too big.  Remember, you still have to add pads too.

 

 

8) Layout Verification:

 

Design Rule Check

From the Session palette select ICrules, then select Check from the ICrules palette (you can get back to that palette by hitting the right mouse button in the palette box and selecting root). A prompt box will appear at the lower left of the screen. Click on OK to proceed with the check.

 

When the check is complete, design rule errors which exist in the layout will be reported in the message bar at the bottom of the ICstation window. If the number of ‘results’ is not zero, then there are errors. The first one can be shown by selecting "First" in the palette. Then rest can be viewed by clicking on "Next".

 

One common error is the bad port error. The port (metal1.port or metal2.port) must be completely covered with the corresponding metal (e.g. the metal 2 interrcontect coming up to a metal 2 port must completely cover the port, not just butt up against it). For some reason, the tool is not doing this and so we need to fix it manually. Go back to the IC Palettes menu and select Easy Edit.  Type “sho la p” in the window to bring up the Show Layer Palette dialog box. Select the layers that you need (metal1 and metal2 in this case) by holding down the ctrl key and clicking on the layer and OK the dialog. The selected layers will appear above the Easy Edit palette. Now, click on the layer you want and select Shape. OK the dialog box that appears and draw required shapes to fix all the DRC errors.  Once you’re finished, run the design rule check again.

 

Layout Versus Schematic(LVS) Check

Verify the layout for consistency with the transistor level schematic of your design. Do this by returning to the main Session palette and selecting the ICtrace(M) (mask level LVS) option. Click on LVS in the ICtrace(M) palette and in the form which appears, select the LVS viewpoint using the Navigator button. For example, if your design is named “ralu” the LVS viewpoint can be found by navigating down into the ralu directory and selecting the LVS directory. Click on the Setup LVS button and in the form that comes up change the following items and click OK

 

Ground Names: VSS GND      Recognize Gates: Yes

 

OK the LVS dialog. When the check is complete, the bottom message bar will read "Mask results database loaded". To view the results, you can select Report -> LVS from the palette menu. Look for the smiley face. If a nasty X appears instead, something is wrong. Find it and fix it.

 

We are done with IC Station for a bit now... you can save your cell and exit if you like.

 

9) Pad Frame Generation

 

The pads are generated in Design Architect (da).  Start DA (in your netlist/work directory after running sul and swd)

 

Symbol Creation

Load the symbol that defines your cell.  The edif2eddm script has already created this symbol for you.

 

Open the symbol from the session palette in DA and then select the symbol body. Then from the right-mouse pull-down menu select Properties>Add > Add Single Property dialog box.  In the form that comes up, specify:

 

Property Name phy_comp

Property Value design_name

 

and place the text inside the symbol where design_name is the name of your design (e.g. ralu). The word “phy_comp” informs DA that this is a physical component.  The property value informs the tools where to find the layout associated with the component design_name  (with respect to $MGC_WD). Check and save the symbol using the Check>With Defaults and File>Save Symbol menu items.

 

Place and connect Pads

Now we have to create a new component/schematic consisting of the core logic and the I/O pads. This is a laborious task.

 

Create a schematic by hitting the “Open Sheet” button in the session palette.  In the form that comes up, enter the name of your component, for example P_ralu.  The component name must be different than the name of your component with your logic in it.  The sheet name must always be sheet1.

 

In the new schematic sheet, instantiate a copy of your core logic using the Choose symbol in the schematic palette and using the navigator to choose the appropriate symbol name.

 

To place pads about your core logic, select  Libraries à ADK Libraries à Pads à AMI 0.5 from the main menu to bring up the ADK pads palette.  Select the pads you need (in, out and power) and place them on the schematic about your core logic as shown below. Wire the core logic and pads together.  Vdd and GND are only connected to ports as shown.  If you don’t remember how to wire up parts, create ports, etc, you can review the instructions at http://web.umr.edu/~daryl/classes/cpe311/W2003/mentor_tutorial_1a.pdf and http://www.ece.umr.edu/courses/cpe214/faq.html.

 

Change the property ‘PINXX’ using the shift+F7 key to replace the ‘XX’ with the pin number. Consider the following pin diagram when naming pins, so that pins and ports are close to one another (e.g. if the port for CLK were on the lower left of the layout, then pin 25 would be a much better choice than pin 40).

 

 

Check and save the schematic when you’re done.  Exit DA.

 

Create Viewpoint

Use the adk_dve script to create viewpoints for this design.

>adk_dve P_designname

10) Layout of the complete chip

Now you’re ready to ready to wire everything up in ADK.  Start ADK by typing the command “ic” at the command prompt (after typing sul and swd).

 

First, we’ll create a cell named P_designname using the layout viewpoint. Do this as you did in step 6: From the Session palette, select ‘Create’ under the “cell” heading and enter the following in the dialog box that appears:

            Cell Name: P_design_name

            Attach Library: $ADK/technology/ic/ami05

            Process: $ADK/technology/ic/ami05

            Rules File: $ADK/technology/ic/ami05.rules

            Angle Mode: 45

            Connectivity? : With Connectivity

            EDDM Schematic Viewpoint: P_design_name/layout

            Logic Loading Options…..

                                    Logic Loading: Flat

 

Go to the ADK Edit palette menu and click on Open to open the logic source window. You should see your schematic in a new window.

Select the core logic (if you have more than one piece, select them all). Then click on Inst on the DLA Logic palette to place these cells into your layout window.  Remember, you must have previously designed these cells and they must have the name of the phy_comp property on your symbol.

 

From the pull-down menu select Setup à SDL and verify that the Search Path looks like this:

$ADK/technology/ic/ami05_via .

The last ‘.’ is important for it includes the current, or working directory. You should add any other paths you wish to search for your cells.

 

Padframe Generation

Once you have placed the core logic block(s), you are ready to generate your padframe. Do not place the pads yourself. From the pull-down menu select

ADK à Generate Padframe à 0.5 micron tinychip

This will generate the padframe.

 

Make life easy on yourself. NEVER EDIT OR RELOCATE PAD CELLS.

 

View the entire cell to see the padframe and your core cell(s). You should see the entire padframe with corner pads, the pads you asked for and the spacer pads for all the empty spaces. You will also see overflows showing which ports get wired to which pins. At this point you might notice that your core logic should be rotated, flipped or moved to make routing easier. Do that, now, but be careful not to move the pads. After finalizing the placement of your core logic, you can route the pads to the core.

 

You can route the pads to the core manually or automatically. To autoroute, click on P&R on the ADK Edit palette menu to bring up the Place & Route palette menu. Then click on All under Autorou. On the prompt-bar that appears click Options and unselect Expand Channels from the menu. Click OK on the menu and prompt-bar to autoroute the pads.

 

Now take a look at the final layout and fix any problems. For example, you might not like the width or placement of the power busses. You can edit these routes (or opt to manually route them, initially) until you are satisfied with the final result.  Don’t forget to run the DRC check if you change anything.

 

Save your cell when you’re done.

11) Preparing Fabrication Data

The ICLink feature will be used to create the GDSII file.

 

Within IC Station, create a new cell with No Connectivity (polygon editing) (Use the File>Cell>Create menu option.  Enter in an appropriate cell name, like “final” and hit the no connectivity option). Make sure you specify the correct process for the cell (in our case, the ami05 stuff as before).

 

Use the right-mouse menu option Add à Cell to add your top-level design to this new cell (Simply type the name of your cell into the dialog box.  If you’ve forgotten the name, you can use the File>Open>Cell menu option and navigate to the correct cell). This top-level should include your wired padframe as well. You should not do any editing of your cell that this point.

 

Use the ADK à Scale menu to select the technology you are targeting. This will scale your design into physical design units (e.g. microns).

 

Save your cell (File>Cell>Save Cell).

 

Use Translate à ICLink to write a GDSII stream (or CIF but MOSIS prefers GDSII and it will scale properly with no manual edits). Select the source format as ICgraph and the destination as GDSII. Click OK and another dialog box will appear that will ask for the source and destination names and option file.  Enter the name of your source cell (or use the navigator), the destination (including any extension such as .gdsii) and the option file. For the option file, you should use $ADK/lib/iclink_gdsii.options.

 

If everything went well, you should now have a GDSII file ready to send to the fabrication warehouse.

 

If there are any errors, the tool prompts you to check the transcript and the ICLink log file. Try to fix any errors, if possible. One common warning in the transcript is about certain Cells not being dynamically loaded. Another is that your device has unplaced objects.  This appears as an error in IC Station but as a warning in the ICLink log file.  As far as we can tell, this is not a problem (???).
