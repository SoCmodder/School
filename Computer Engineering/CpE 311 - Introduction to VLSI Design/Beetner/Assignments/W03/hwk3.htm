  1. Using the following procedure, construct a VHDL model of a two input NAND gate with an entity equal to the following:
   ENTITY nand2 IS
     PORT (a,b: IN BIT; c: OUT BIT);
END ENTITY nand2;

1.                 This assignment should typically be run from your ece Unix account. If a Unix machine isn't available, you can run Modelsim remotely from an NT machine using X-WIN32. You can start X-WIN32 from the START>Computer Center Applications>Network Access>X-Win32. Once started, you can open a session to one of the ECE unix machines by pressing the X-Win32 button on your menu bar and selecting the appropriate machine from the Sessions menu. If the machine you want is not there, you can create a new session.  Check out my Xwin32 hints document. 

2.                 If you need help using Mentor Graphics, check out the users guide for Modelsim. The file /usr/local/modeltech/docs/ee_doc_index.pdf is a good place to start. Use acroread on the Unix machines to read it (i.e. acroread ee_doc_index.pdf).

3.                 Create a new subdirectory in your Unix account called twoinand.

4.                 Use your favorite text editor to create a source file called my_nand.vhd and enter the entity shown above.

5.                 Create a work directory with the command: vlib work. You only need to do this once. This creates a subdirectory in twoinnand which will persist until you delete it. Work is where the VHDL analyzer creates all of its output files.

6.                 Compile the source file with the command: vcom -93 my_nand.vhd. This compiles using VHDL'93 syntax. Leave off the 93 for VHDL'87 syntax. Enter vcom -help for a list of options.

7.                 Examine the work directory with the command vdir. You should see the name of the entity you just analyzed.

8.                 Use your text editor to add a suitable architecture to your nand model. Call it A1 and use a simple signal assignment statement for the model.

9.                 Compile the source file again. Fix any errors.

10.             Enter the command vdir nand2 and describe the result.

11.             Bring up the simulator with the command vsim nand2 a1. This will invoke the simulator on the entity nand2 and the architecture a1 and start a command window.

12.             In the command window, type the command view * and describe each of the windows displayed.

13.             In the command window, type the command add wave * and describe the signals which are displayed.

14.             Type the command force a 0 to force signal a to a zero. Type a similar command for input b.

15.             Run the simulator for 100 nS by typing the command run 100.

16.             Force a to a 1 and run for another 100 nS.

17.             Force b to a 1 and run for 100 nS.

18.             Try clicking around in some of the windows or selecting signals and hitting the right mouse button. Play around a bit. Describe some interesting things you found that you can do with the simulator.

19.             Finish the grey code input sequence ab=00,10,11,01,00. You can force multiple values using the command force a 0 0, 1 10, 0 20 where value/time pairs are separated by commas. Your simulation should last for 500 nSec.

20.             Make a plot of the wave window and exit the simulator (you can make a plot by using the FILE>Write to Postscript option in the wave window).

Draw a block diagram for the circuit that would likely be synthesized for the following VHDL code.  Clearly label any signals/wires appropriately, especially circuit inputs and outputs.  Draw boxes around entities or “entity-like” structures.  Label components appropriately if not obvious (e.g. “flip-flop”, “OR-gate”, “adder”, etc.)

library IEEE;

use IEEE.std_logic_1164.all;

 

entity e1 is

    port(a,clk: in std_logic;

        f:out std_logic);

end entity e1;

 

architecture a1 of e1 is

    signal b: std_logic;

begin

    p1: process (a,b) is

    begin

        f <= a AND b;

    end process;

    p2: process (clk) is

    begin

        if (clk = '1') then

            b <= a;

        end if;

    end process;

end;

·      Inputs: D, CLK, CLR, and PRS (all single bits)

·      Outputs: Q, Q_not

CLR (clear) and PRS (preset) are asynchronous. Clear sets the output to ‘0’ and preset sets it to ‘1’. At a minimum, compile your design using VCOM to be sure it is syntactically correct.
