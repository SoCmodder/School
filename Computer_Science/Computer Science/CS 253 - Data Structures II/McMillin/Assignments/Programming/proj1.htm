CSC 253 Programming Assignment 1


Fall; 2003 (Due 9/25/2003) at the beginning of class

Be sure to read the Programming Style document when turning in your assignment. You must work in a small group for this assignment.

In this lab we will use the actual machine to validate our run-time complexity analysis and asymptotic run-time complexity analysis by comparing worst case, best case, and average case run times for Insertion Sort, Bubble Sort, and Merge Sort.  You should implement all three algorithms, then plot their time behavior as a function of n and compare this with plots of the expected behavior.

 

Some questions to think about:

 

  • What size n0 is required to begin to exhibit asymptotic complexity?
  • What inputs are required to generate “average” complexity.  How about “best case and worst case”?  Are the inputs the same for all algorithms?  If not, is this a fair comparison?
  • How does measured run time correspond to operation counts that we use in abstract complexity analysis.  Are you counting the same thing?

 

To be effective in this assignment you should read and understand Chapters 2 and Section 3.1 thoroughly.
