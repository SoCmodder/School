Exercise 2.1-3

Exercise 2.1-3 p.21

PROBLEM:

Consider the searching problem:

INPUT: A sequence of n numbers A=< a_1, a_2,...,a_n > and a value v.
OUTPUT: An index i such that v=A[i] or the special value NIL if v does not appear in A.

Write pseudocode for linear search, which scans thru the sequence, looking for v. Using a loop invariant, prove that your algorithm is correct. Make sure that your loop invariant fulfills the three necessary properties.

SOLUTION:

Your loop invariant will depend on your pseudcoded implementation. Below are two solutions for two slightly different linear search implementations (you may find solution 2 easier to follow) and their corresponding loop invariants:

Solution 1:

Postcondition: if v was found in A, then i exists such that A[i]=v, and if v was not found in A, then i exists such that i==NIL. (This postcondition is just a rewording of the OUTPUT specification in the problem statement).

Pseudocode: (whether this is pseudocode, or real code is not important) 

 
 1      #define NIL 0
 2      i=NIL;
 3      j=1;
 4                                  // assert {I}
 5      while(j<=n AND i==NIL)
 6      {
 7                                  // assert {I} AND (j<=n AND i==NIL)
 8        if (A[j]==v)
 9          i=j;
10        else
11          j++;
12                                  //assert {I}
13      }
14                                  // assert {I} AND NOT(j<=n AND i==NIL)

This program achieves the postcondition by maintaining the following loop invariant:

{ I }:   A[i]=v OR (i=NIL AND A[k]!=v for k=1..j-1)


To convince ourselves that this loop invariant is valid, we must show two things:

    1. Assert that this invariant holds true on lines 4, 7, 12, and 14 (which corresponds to the INITIALIZATION, MAINTENANCE, and TERMINATION phases). If these assertions hold true, then we must go on to step 2...
    2. Confirm that the invariant implies the postcondition when the loop terminates (in other words, if the invariant holds true upon TERMINATION, then it should also be true that the Postcondition is satisfied). (Why is step 2 necessary? See the NOTE below).


In order for you to learn why this invariant works, you really should go thru and do all the assertions by hand. Believe me when I say you will not remember anything about Loop Invariants unless you process (on paper) the theory by yourself!

NOTE:
Why is step 2 necessary? Because, we could create an invariant that was always true, for example:

{ I } :   Penguins don't fly.

Note that with this invariant, all our assertions would pass (because the invariant is always true), but during the TERMINATION phase when we ask whether the true invariant implies a satisfied postcondition, the answer would be no, it doesn't. The fact that Penguins don't fly says nothing about the final status of i.

Solution 2:

Postcondition: If v was found in A, then i exists such that A[i]=v. If v was not found in A, then i exists such that i==NIL AND v!=A[j] for j=1, ... ,n.

Pseudocode: 

 
 1      i=1
 2      while (A[i]!=v AND i<=n)
 3      {
 4        i++;
 5      }
 6      if (i > n)
 7      {
 8        i=NIL;
 9      }

This program achieves the postcondition by maintaining the following loop invariant:

{ I }:   A[i]=v OR (A[j]!=v for j=1..i-1)

Note that this invariant doesn't say anything about i ever being NIL. This is okay since our loop also doesn't say anything about i ever being NIL. The i=NIL assignment occurs after the loop has terminated. The loop invariant doesn't need to include information about what happens outside the loop. To convince ourselves that this loop invariant is correct, we must go thru the two steps shown in Solution 1. This validation is left to the reader.  Note, also, that, as in class, the A[i]=v term is not really needed; it is given to us by the loop guard.
